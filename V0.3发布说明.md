# C语言代码理解助手 V0.3 发布说明

## 版本定位

`V0.3` 是这一产品线从“代码讲解器”向“改题辅助器”迈出的第一步。

前面的版本已经能解释：
- 这段代码用了什么语法
- 这段代码大概在做什么
- 更像对应哪类赛题要求

这一版开始进一步回答：
- 这段代码依赖谁
- 改这里会影响哪里
- 这段代码的执行链是什么
- 哪些改法最容易把模板改坏

## V0.3 解决了什么问题

真实场景里，新生往往不是完全看不懂代码，而是：

- 看懂了这段代码“属于参数逻辑”
- 但不知道它依赖哪些变量和函数
- 不知道改一个字段会不会把保存、读取、显示一起带坏
- 不知道应该先改输入、处理中间量，还是最后输出

`V0.3` 重点就是解决这些问题。

## 本次新增能力

### 1. 这段代码依赖谁

现在会直接提示：
- 依赖哪些变量
- 依赖哪些函数
- 依赖哪些外设接口

例如：
- 依赖 `g_param`
- 依赖 `AT24C02_WriteByte`
- 依赖 `Key_GetEvent`

### 2. 改这里会影响哪里

现在会直接提醒：
- 改参数结构体，可能影响保存 / 读取 / 显示
- 改页面编号，可能影响按键切页 / 显示函数
- 改显示位，可能影响数码管布局
- 改采样变量，可能影响显示页和报警判断

### 3. 代码执行链

现在会把一段代码拆成：
- 输入是什么
- 中间处理了什么
- 最后输出到哪里

这样 `App_SaveParam`、`App_HandleKey`、`App_ShowDataPage` 这类代码看起来就会更不一样。

### 4. 高风险改动提醒

这版会更主动提醒这些最常见的坑：
- 结构体字段改了，但保存逻辑没同步
- 页面枚举改了，但切页逻辑没同步
- 变量改名了，但显示/判断还在用旧名字
- 改了显示位，字符位和小数点没同步

## 当前适合谁用

- 大一新生
- 刚开始接触模板工程
- C 语法基础还不稳
- 已经开始尝试自己改页面、按键、参数和显示逻辑的人

## 当前版本建议

`V0.3` 已经适合继续给新生和客户试用。

这版最适合拿来做的事情不是“学完整个 C 语言”，而是：
- 看懂模板
- 定位代码作用
- 降低改题时只改一半的风险
