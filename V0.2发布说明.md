# C语言代码理解助手 V0.2 发布说明

## 版本定位

`V0.2` 是在 `V0.1` 试用版基础上的一次重点体验迭代。

这次迭代不是简单加功能，而是专门针对真实客户反馈做优化：

> 右侧讲解太宽泛，不同代码看起来讲得都差不多。

所以 V0.2 的目标很明确：

- 让不同类型的代码，讲出来更像不同类型的代码
- 让新生能从右侧结果里更快看出“这段代码到底在做什么”

## 这次主要更新了什么

### 1. 新增“这段代码具体做了哪些动作”卡片

现在不再只告诉用户：

- 这是按键逻辑
- 这是参数逻辑
- 这是显示逻辑

而是会继续往下讲：

- 这段代码在做 EEPROM 写入
- 这段代码在读按键值
- 这段代码在切页面
- 这段代码在决定数码管显示
- 这段代码在更新温度 / ADC / 频率 / 距离 / RTC 数据

### 2. 增加关键调用识别

对新生最常见、最有教学意义的动作做了更具体的识别，例如：

- `AT24C02_WriteByte / ReadByte`
- `SEG_SetCode / SEG_SetDigit`
- `Key_GetEvent`
- `DS18B20 / DS1302 / PCF8591`
- 频率、超声波、串口相关调用

### 3. 增加“真正被修改的变量”识别

现在右侧结果不只看函数名，也会看：

- 这段代码到底改了哪些变量

这样会比只看“调用了什么函数”更贴近真实业务含义。

### 4. 优化修改建议

现在“如果你要改这段代码，先看哪里”这部分也更具体了。

例如：

- 保存参数代码，会提示先看参数结构体和 EEPROM 地址布局
- 按键处理代码，会提示先看 key 值进了哪个分支
- 显示代码，会提示先看显示前的数据怎么计算出来

## V0.2 解决了什么问题

这次最核心的变化是：

- `App_SaveParam` 和 `App_HandleKey` 的讲解不再那么像
- 页面显示函数和数据采样函数也会更容易区分
- 新人看右侧结果时，更容易感觉“它真的在讲当前这段代码”

## 当前适合怎么使用

推荐这样用：

1. 把看不懂的一段模板代码粘进去
2. 先看“更像哪类赛题要求”
3. 再看“具体做了哪些动作”
4. 最后看“如果你要改这段代码，先看哪里”

## 当前版本判断

我对这一版的判断是：

- 已经比 `V0.1` 更适合给新生和客户继续试用
- 更接近一个真正的“模板代码讲解工具”
- 后面可以继续根据真实反馈小步迭代，而不是大幅重构
